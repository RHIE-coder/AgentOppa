# route.md — 각 수단에 실제로 주입하는 법

[classify.md](classify.md)에서 범위·수단을 정했으면 여기서 실제로 넣는다. 위(강하고 싼 것)부터.

## 검사기 룰 + 테스트 (기계로 잡힘 — 1순위)
- 어느 검사기? 교훈이 막는 *결과물의 메이커* `scripts/validate.mjs`. 스킬 모양→ccc-skills · 플러그인/결합→ccc-plugin · 하네스→agent-engineer · 메모리→ccc-memory. 이 repo만이면 `test/`.
- 룰 = 짧은 검사 한 조각. Node 기본기능만 · `err/warn/ok` · 종료코드 0/1/2 · 코드블록(```)·인라인코드는 예시라 건너뜀.
- **반드시 테스트 한 쌍** — 빨강(나쁜 예 → 실패) + 초록(정상 → 통과). 교훈을 *말*이 아니라 *테스트*로 주입한다. 테스트는 검사가 훑는 플러그인 *밖*(`.agentoppa/fixtures/`)에 둬 repo를 더럽히지 않는다.
- ⚠ **룰은 범용으로** — 특정 사건 이름(인스턴스)을 넣지 말 것. 그건 로그로.

```text
예) "스킬이 다른 스킬의 examples/ 를 링크하면 → 실패"
   ccc-plugin 검사기에 스킬 글 훑기 추가 (코드블록 건너뜀, 자기 것은 봐줌) + 빨강/초록 테스트.
```

## 훅 (특정 순간 자동 실행 — 평소 컨텍스트 0)
- 특정 *행동*을 자동으로 막거나 검사. 예: 위험 명령 직전 막기, 턴 끝낼 때 검증 자동. `sync=strict`의 게이트 훅이 이 형태. (ccc-hooks)

## 리뷰어 한 줄 (ccc-agents)
- 판단형 교훈을 *리뷰 담당 에이전트 지시문*에 한 줄. 리뷰 돌 때만 작동(서브에이전트라 메인 컨텍스트 0).

## Gate(통과 조건) 한 줄 (그 스킬 ## Gate + sync)
- "끝내기 전 반드시 확인"인 판단형. 세기 = `sync`: loose(안내)·medium(경고)·strict(ccc-hooks가 막는 훅 생성).

## 문서 한 줄 (그 컴포넌트 references/)
- 그 결정을 *맡은* 컴포넌트의 `references/`에 한 줄. 그 작업 읽을 때만 로드된다.

## 항상 켜진 규칙 (배포=always-on.md · 내부=AGENTS.md)
- 어디에도 안 맞고 *늘* 살아야 하는 한 줌만.
- **범위로 자리가 갈린다:** 유저에게도 적용 = `plugins/agentoppa/always-on.md`(세션 주입; 우리 dev도 `--plugin-dir`로 받음) / 이 repo만 = `AGENTS.md`. (ccc-memory가 크기를 검사로 강제 + Claude는 `CLAUDE.md @AGENTS.md`로 같은 걸 읽음.)
- **크기 상한:** 넣어서 한도 넘으면 하나를 검사기로 옮겨 빼거나 둘을 합쳐라(무작정 추가 금지).
- ⚠ *검사기로 잡는 건 여기 또 적지 마라* — 검사기가 원본이다.

## 주입 뒤 — 비우기 + 로그
- **산출물**은 정한 자리에 — 메모 더미 만들지 말 것(가드는 *그게 필요한 자리*에).
- **경위·추론**(무슨 일·왜·일반화·수단·기대)은 `.agentoppa/hardening-log.md`에 한 엔트리로 남긴다(append-only). 이건 *세션에 로드되지 않는 아카이브*라 컨텍스트 0 — `mistakes.md`식 상시 메모와 다르다.
- 후보 큐(`.agentoppa/candidates/`)는 비운다.
