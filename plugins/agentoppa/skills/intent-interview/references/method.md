# method.md — 면담 방법 (분야 무관)

면담의 *방법*만 담는다. 무엇을 만들어 어떻게 넘기는지는 [handoff.md](handoff.md), 큰 흐름은 [SKILL.md](../SKILL.md). 이 파일은 분야를 안 가리는 *방법* — 무엇을 캐는지의 분야별 디테일은 그때그때 정한다.

## 확신은 어떻게 쌓나

AI는 진짜 확률을 못 잰다 → *대신 쓰는 신호*로 확신을 쌓는다. 확신이 *진짜 오르는* 신호와 *가짜인* 신호를 구분하라:

| 확신이 오른다 (진짜) | 확신이 가짜다 |
|---|---|
| 사용자가 직접 확인한 구체 답 | 내가 "아마 이걸 원하겠지" 짐작 |
| 구체 예시/반례로 못박힘 | "좋게/잘/적당히" 같은 두루뭉술 답 수용 |
| 되짚어 확인했더니 정정 없음 | 침묵을 동의로 여김 |
| 둘 중 하나 고르라니 골랐음 | 둘 다 좋다는 답을 그대로 둠 |

**한 줄 테스트:** "지금 이대로 넘기면, 이 대화를 못 본 사람이 사용자가 *진짜* 원한 걸 만든다." 여기에 자신 있게 "예"면 ~95%.

## 무엇을 캐야 확신이 단단한가

각 항목에 *사용자가 확인한 구체 답*이 있어야 확신이 선다. 빠진 항목 = 다음 질문 후보:

- **결과(목표)** — 기능이 아니라 결과를 묻는다. "그게 되면 *뭐가 달라지나*?"
- **범위 / 안 할 것** — 할 것과 **안 할 것**. 안 할 것이 *말과 속내의 차이*를 가장 잘 잡는다. "이건 *아닌* 거죠?"
- **제약** — 도구·시간·기존 것·꼭/절대. "이미 쓰는 것, 못 바꾸는 것은?"
- **예시 / 반례** — "좋아하는 사례 하나, 싫은 사례 하나." 두루뭉술을 구체에 묶는 가장 빠른 길.
- **우선순위** — "A와 B 중 하나만 된다면?" 강제로 고르게 하면 진짜 순위가 나온다.
- **성공 기준** — "다 됐다는 걸 어떻게 알까?"

> 하네스를 설계하는 자리면 더한다: 어떤 작업 묶음(기획·개발·리뷰·병렬?) · 분야 · `sync` 세기(안내만? 경고? 막기?) · 병렬 필요 여부 · 모델 등급. 이 값들이 곧 `config.yaml`이 된다.

## 좁히기 vs 넓히기 — 신호로 고른다

| 사용자 신호 | 모드 | 방법 |
|---|---|---|
| 구체적·확신·디테일을 먼저 말함 | **좁히기** | "둘 중 뭐냐" 질문 · 빈칸 채우기 · 되짚어 확인 |
| 두루뭉술·"몰라"·"알아서"·모순·침묵 | **넓히기** | ↓ 넓히기 방법들 |

한 면담 안에서 모드는 계속 바뀐다 — 큰 그림은 넓혀서 열고, 잡히면 좁힌다.

## 넓히기 방법들 — 두루뭉술한 사용자 (상담사 모드)

**두루뭉술한 사용자에게 "뭘 원하세요?"를 반복하지 마라** — 그건 *답할 수 없는* 질문이다. 답을 *만들게* 하지 말고 *고르게/반응하게* 하라:

- **양 극단 두 개 제시** — "딱 필요한 것만 가볍게? 아니면 빈틈없이 꼼꼼하게?" 가운데를 고르면 다시 두 극단으로 좁힌다.
- **임시 안 던지기** — 그럴듯한 초안을 *일부러 던지고 반박을 줍는다*. "기본으로 이렇게 가면 어때요?" → "아니 그건…"에서 진짜 속내가 나온다.
- **예시·비교** — 아는 사례로 디딤돌을 놓는다. "X처럼? Y처럼?" 두루뭉술한 말을 구체 사례로 바꾼다.
- **상황 그려보기** — "이런 상황이 오면 어떻게 동작하길 원해요?" 말 대신 *행동*으로 속내를 끌어낸다.
- **조금씩 구체화** — 큰 갈래 → 작은 갈래 → 구체. 한 번에 바닥까지 안 내려간다.
- **반응 캐기** — 좋다/싫다/"음…"의 *이유*를 캔다. 그 이유가 다음 질문을 정한다.

## 되짚어 확인 (내가 이해한 걸 다시 말하기)

가끔 모은 그림을 요약해 사용자에게 되읽어 준다 — "제가 이해한 건 …, 맞나요?"

- **왜:** (가) 확신을 *재고*, (나) 정정을 끌어내 말과 속내 차이를 드러내며, (다) 사용자가 자기 생각을 객관적으로 보게 한다.
- **언제:** 큰 결정 직후 · 넘기기 직전엔 반드시. 정정은 실패가 아니라 *수확*이다.

## 하지 말 것

- **질문 몰아치기** — 한 번에 여러 개 → 머리 아프고 답이 얕아짐. 항상 하나씩.
- **알 수 있는 걸 묻기** — 추론 가능한 걸 물으면 신뢰가 깎인다. 먼저 추론하고 *확인만* 받아라.
- **두루뭉술 답 수용** — "좋게요" 받고 넘어가기 ❌. 구체 예시/반례로 묶을 때까지 머문다.
- **확신 전에 넘기기** — 위 4가지 체크가 다 차기 전엔 멈추지 않는다.
- **너무 캐묻기** — 사용자가 가려는데 계속 붙잡기. *덜 중요한* 미지수는 "남은 질문"으로 넘기고 멈춰라.
- **답 유도** — 원하는 답으로 몰기. 선택지는 한쪽으로 기울지 않게, 극단은 양쪽 다.
- **넓히기만 하고 안 좁히기** — 넓힌 뒤엔 **반드시** 좁혀서 모은다.
